## はじめに

本プロジェクト（旧 multi-agent-shogun）は、現在戦国テーマで運用されています。
本計画書は、これを **FINAL FANTASY XV (FF15)** の世界観、特に **ルシス王国** を中心としたテーマに全面改修するための包括的な戦略文書です。

改修の主な目的：
1. **テーマの統一**: 戦国テーマからFF15/ルシス世界観への完全移行
2. **キャラクター・役職の統一**: Noctis, Ignis, Kingsglaive への完全刷新
3. **ドキュメント・コード・コメントの改修**: 全ファイルの一貫性確保
4. **プロジェクト構造の刷新**: フォルダ名・セッション名のFF15テーマ化（multi-agent-ff15）

## 第1部: 世界観再定義（基礎設計1）

### 1.1 政治体系

**ルシス王国** を中心とした世界観：

- **統治者**:
  - 国王レギス・ルシス・チェラム（Regis Lucis Caelum）
  - 王妃アリシア（Aulea）
  - Noctis（Noctis Lucis Caelum）- 本プロジェクトの指揮官（Shogunから改称）

- **主要施設**:
  - 王都インソムニア（Insomnia）- 魔光壁に守られた王国の中心
  - 王城 - 指揮統制の本部

- **外交関係**:
  - **敵対勢力**: ニフルハイム帝国（魔導兵と魔法技術で武装）
  - **同盟国**: イェンシマ、テネブラエ（王妃アリシアの故郷）

### 1.2 魔法・魔導兵体系

**六神とクリスタルの関係**:
- バハムート（龍神王）、シヴァ（氷女神）、ラムゥ（雷神）、イフリート（炎神）、リヴァイアサン（海女神）、タイタン（土神）
- クリスタル: 古代の遺産で、王族の血統にのみ扱える力源。六神と召喚獣を制御

**魔導兵**:
- ニフルハイム帝国が開発した魔法と機械の融合兵器
- MT（Magitek Trooper）として歩兵・車両・航空機で運用

### 1.3 社会構造

**王都インソムニアの階級体制**:

| 階級 | 役割 | 本プロジェクトへの対応 |
|------|------|----------|
| 王族 | 最高権力、クリスタル管理 | Shogun（将軍）→ Noctis |
| 側近・軍師 | 王族の補佐・計画立案 | Karo（家老）→ Ignis |
| 王の剣 (Kingsglaive) | 実働部隊・特殊任務 | Ashigaru（足軽）→ Kingsglaive |

4. **Kingsglaive 4**: イリス (Iris) - 支援・ロジスティクス
5. **Kingsglaive 5**: コル (Cor) - 不死将軍、実戦指揮・ベテランの知見
6. **Kingsglaive 6**: アラネア (Aranea) - 傭兵、空中戦・外部協力
7. **Kingsglaive 7**: レイヴス (Ravus) - 帝国軍将軍、戦略的視点・武力

## 第5部: 実行計画

### 5.1 フェーズ分け

#### **フェーズ1: 基礎設定確認 & リスク評価**（1日目）
- **目的**: 改修対象ファイルを確定し、リスクを把握
- **タスク**:
  1. 全ファイルのバックアップ取得
  2. 改修対象ファイルの詳細スキャン
  3. 置換パターンの詳細設計
- **成果物**: リスク評価レポート + 置換パターン辞書

#### **フェーズ2: instructions/*.md 改修**（2-3日目）
- **目的**: 役職別指示書をFF15テーマへ改修
- **タスク**（並列投入）:
  1. instructions/shogun.md の改修（Kingsglaive 1）
  2. instructions/karo.md の改修（Kingsglaive 2）
  3. instructions/ashigaru.md の改修（Kingsglaive 3）
- **工数**: 各ファイル 1-2日
- **成果物**: 3つの改修済み instructions ファイル

#### **フェーズ3: AGENTS.md 改修**（4日目）
- **目的**: プロジェクト全体の概要書をFF15テーマへ改修
- **タスク**:
  1. AGENTS.md の全セクション改修
  2. アーキテクチャ図・フロー図の刷新
- **工数**: 1-2日
- **成果物**: 改修済み AGENTS.md

#### **フェーズ4: 検証 & デプロイ**（5日目）
- **目的**: 全改修を検証し、本番へ反映
- **タスク**:
  1. 全ファイルの一貫性確認
  2. 既存テストの実行
  3. 新しい instructions で実際にエージェント起動テスト
  4. git への commit & push
- **工数**: 1日
- **成果物**: デプロイ完了、git history に改修履歴が記録される

### 5.2 工数見積

| フェーズ | 主要タスク | 工数 |
|---------|----------|------|
| 1 | バックアップ + 設計 | 0.5日 |
| 2 | instructions 3ファイル改修（並列） | 2日 |
| 3 | AGENTS.md 改修 | 1.5日 |
| 4 | 検証 & デプロイ | 1日 |
| **合計** | | **5日** |

### 5.3 リスク管理

| リスク | 対策 |
|--------|------|
| ファイル編集ミスによる構文エラー | 各ファイル改修後、即座に LSP diagnostics 実行 |
| 置換漏れによる古い用語の残存 | 全ファイルを正規表現でスキャンし、残存を確認 |
| エージェント起動失敗 | 改修後、実際にエージェントを起動テスト |
| git conflict | 改修を細かい commit に分け、順序を明確化 |

### 5.4 成功基準

- [ ] 全 instructions/*.md ファイルが FF15 テーマで統一されている
- [ ] AGENTS.md が FF15 プロジェクト概要書として機能している
- [ ] エージェント起動時にエラーが発生しない
- [ ] 既存テスト（存在する場合）がパスする
